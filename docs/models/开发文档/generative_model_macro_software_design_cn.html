认知导航模块软件设计说明

基线文档：generative_model_macro_framework_cn.html，generative_model_macro_requirements_cn.html

本文档面向当前宏观数学框架的软件落地，说明系统分层、模块职责、核心数据结构、 算法流程、校验与异常策略、输出 artifact 和实现顺序。设计核心是 kernel-first： 三类表征只在 R_M -> P_M 阶段分化， 得到 row-stochastic transition kernel 后，共享 cognitive graph、distance、MDS、A* 和 edge expansion 编排流程。

1. 文档目的

本文档描述如何把当前数学模型和需求分析转化为可实现、可测试、可审计的软件结构。 它不依赖任何现有代码实现细节，也不规定最终必须采用某个具体文件名或第三方库； 但会给出推荐的模块边界、数据结构、接口契约和算法步骤。

设计目标包括：

  • 把 K_0、R_M、P_M、cognitive graph、distance、embedding、A* 和 expansion 表达为明确的软件对象。
  • 把 primitive、action-chunk、landmark 的差异限制在 representation、kernel construction 和 edge expansion 层。
  • 保证每个 pipeline stage 可以单独验证，并能输出足够的中间 artifact 支撑调试和测试。
  • 让正向生成流程在失败时返回可识别、可定位的错误，而不是产生静默错误结果。

2. 设计原则

原则 设计含义
Kernel-first transition kernel 是统一接口。后续图、距离、嵌入和规划逻辑只依赖 kernel，不直接依赖表征内部字段。
Stage isolation 每个阶段输入输出清晰，不把校验、kernel 构造、图算法、搜索和展开混在同一个函数中。
Explicit provenance 每个 kernel edge、A* edge 和 expansion step 都能追溯来源，便于核查概率、cost 和采样结果。
Deterministic by configuration 所有随机采样由显式 RNG 控制；所有 tie-breaking 采用固定顺序并记录到 metadata。
Fail explicitly 输入非法、概率不归一、distance 不有限、goal 不可达或 expansion 不可执行时，返回明确错误或状态。
Small graph clarity 当前研究任务规模较小，优先使用清晰直接的算法；性能优化必须保持数学语义不变。

3. 设计范围

3.1 输入对象

  • 确定性 primitive task graph：状态、动作、合法动作、转移函数和 primitive cost。
  • 三类静态表征：primitive action weights、action-chunk token weights、landmark CPD blocks。
  • 可选表征构造参数：用于从 task 和显式规则构造 primitive、chunk 或 landmark representation。
  • trial 描述：起点、目标、表征类型、可选随机种子。
  • 运行配置：数值容差、MDS 参数、A* heuristic scale、tie-breaking、路径枚举上限。

3.2 输出对象

  • 模型 artifact：transition kernel、kernel support、cognitive graph、directed distance、symmetric distance、embedding。
  • 规划 artifact：A* trace、cognitive path、path cost、search diagnostics。
  • 执行 artifact：edge expansion trace、primitive actions、observed states、提前终止记录。
  • 运行 metadata：输入标识、配置、随机种子、状态、失败阶段与失败原因。

3.3 边界

  • 本文档只设计正向生成 pipeline，不设计参数学习、反推、交互式可视化或实验数据清洗流程。
  • RepresentationBuilder 可以根据 task 和显式参数构造表征，但不负责从数据中学习或推断表征参数。
  • 二维 embedding 是 artifact 和 heuristic 来源，A* 搜索图始终是由 transition kernel 生成的 cognitive graph。
  • landmark 被设计为 concrete state set；CPD blocks 是 landmark kernel 的唯一概率来源。

4. 总体架构

系统采用分层 pipeline 架构。上层编排负责把阶段串起来，下层模块只处理自己的输入输出。

Input TaskGraph、Representation 或 RepresentationSpec、TrialSpec、Config。
Representation 可选：由 TaskGraph 和显式参数构造静态表征。
Validation 结构合法性、概率归一化、可执行 token、CPD blocks。
Kernel family-specific builder 生成 TransitionKernel 与 KernelSupport。
Geometry cognitive graph、cost、directed distance、symmetric distance、MDS。
Planning A* 在 cognitive graph 上输出 cognitive path。
Expansion 按表征类型把 cognitive edge 展开为 primitive trajectory。
Output RunResult、artifacts、trace、metadata、可序列化结果。

4.1 推荐包结构

初版实现采用认知功能导向的模块命名。包名使用 cognitivemap.cognitive_navigation， 表示该模块围绕认知表征驱动的导航、规划和轨迹展开，而不是围绕某个实现阶段命名。 

cognitivemap.cognitive_navigation/
  __init__.py
  types.py               # 数据模型、枚举、配置、错误类型、序列化 helper
  task.py                # TaskGraph、primitive execution、shortest paths
  representations.py     # 三类 Representation、RepresentationBuilder、token 查询、表征校验
  kernel.py              # TransitionKernel 与 family-specific builders
  cognitive_graph.py     # cognitive edge、-log(P) cost、distance、MDS
  planning.py            # A* 搜索、edge expansion、执行 trace
  agent.py               # CognitiveNavigationAgent、统一运行入口、artifact 输出
这里的 cognitive_graph.py 不是通用图工具模块，而是当前数学模型中的认知图层： P_M -> E_M -> c_M -> d_M -> D_M -> psi_M。 因此不使用泛化的 graph.py 命名。

4.2 依赖方向

types
  -> task
  -> representations
  -> kernel
  -> cognitive_graph
  -> planning
  -> agent

上游模块不得依赖下游模块。例如 representation 不应依赖 A*，kernel builder 不应依赖 run result。 planning.py 可以调用 expansion 逻辑，但 A* 的核心函数仍然只依赖 CognitiveGraph、 Embedding 和规划配置。

5. 模块设计

5.1 types.py

定义跨模块共享的数据模型、枚举、配置、错误类型和基础序列化 helper。 初版将这些轻量横切对象放在同一模块，避免过早拆出过多薄文件。

对象 职责
StateId 状态标识。推荐使用 str | int，内部统一映射到稳定 index。
ActionId primitive action 标识。
Family primitive、chunk、landmark 三值枚举。
Token tagged union：PrimitiveToken 或 ChunkToken。
Probability 语义类型，表示闭区间 [0,1] 内的浮点概率。
PlanningConfig 数值容差、A* heuristic scale、tie-breaking、MDS 维度和路径枚举上限。
CognitiveNavigationError 所有可识别错误的基类；具体错误作为子类或错误码定义在同一模块。

5.2 task.py

提供 primitive task graph 的构造、只读查询、token execution 和 primitive shortest-path 算法。

  • TaskGraph.from_transitions(...)：从嵌套字典形式的 transition 输入构造完整 TaskGraph。
  • legal_actions(state)：返回状态下合法 primitive actions。
  • next_state(state, action)：执行合法 action 后的确定性目标状态。
  • action_cost(state, action)：返回 kappa(s,a)。
  • execute_token(start, token)：执行 primitive token 或 chunk token，返回完整 action/state trace。
  • shortest_distance(source, target)：返回 primitive shortest-path distance。
  • all_shortest_paths(source, target, limit)：枚举 shortest primitive paths，供 landmark expansion 使用。

正式任务输入以 transitions 为核心。用户只需要显式给出每个 source state 下 每个 action 到达的 target state；states、actions、 legal_actions 和默认 action_costs 由构造函数推导。 这可以自然表示带 loop 等非空间相邻动作的任务图。

5.3 representations.py

定义三类静态表征、状态相关 token 查询和表征校验。

类 核心字段 核心方法
PrimitiveRepresentation action_weights_by_state tokens_at(state) 返回合法 primitive tokens；weight(state, token) 返回权重。
ChunkRepresentation chunk_inventory、token_weights_by_state executable_chunks_at(state, task)；tokens_at(state, task) 返回 primitive tokens 与可执行 chunk tokens。
LandmarkRepresentation landmarks、p_ll、p_lu、p_ul block_for(source, target) 返回 CPD block 类型与概率。

RepresentationBuilder

RepresentationBuilder 是可选工厂，用于从 TaskGraph 和显式参数/规则构造表征。 它可以读取 task 的状态集合、动作集合和合法转移来补齐默认行、过滤不可执行 chunk 或校验 landmark key； 但它不从 task 唯一推断心理表征，也不学习权重或 CPD。 

RepresentationBuilder:
  build_primitive(task, action_weights | policy) -> PrimitiveRepresentation
  build_chunk(task, chunk_inventory, token_weights | policy) -> ChunkRepresentation
  build_landmark(task, landmarks, cpd_blocks) -> LandmarkRepresentation
  validate_compatible(task, representation) -> None
  • build_primitive：根据 task 的合法 action 行构造 primitive action weights；若使用默认 policy，必须明确写入生成规则。
  • build_chunk：根据 task 过滤每个 state 下可完整执行的 chunk，并在 executable token set 上构造或校验权重。
  • build_landmark：校验 landmarks 属于 task states，并校验 LL/LU/UL CPD blocks 与 L/U 集合一致。
  • validate_compatible：只做兼容性校验，不修改 representation。

表征校验与表征对象放在同一模块：primitive 权重、chunk executable token row sum、 landmark CPD blocks 都需要访问表征内部字段，单独拆出校验模块会增加跳转成本。 表征是 task-related 的：它的状态、动作、chunk 和 CPD key 必须能与某个 TaskGraph 对齐； 但表征对象不持有 TaskGraph，只在运行时与传入任务做兼容性校验。

5.4 kernel.py

把任务和表征转换为统一的 TransitionKernel，并并列输出 KernelSupport。

  • build_transition_kernel(task, representation)：按 family 分派到具体 builder，返回 (TransitionKernel, KernelSupport)。
  • build_primitive_kernel(...)：按支持同一 endpoint 的 action 权重求和。
  • build_chunk_kernel(...)：过滤 executable tokens，再按 endpoint 求和。
  • build_landmark_kernel(...)：按 LL、LU、UL CPD block 查表拼接，U-U block 为 0。

kernel builder 必须同时输出 KernelSupport， 记录每条非零 edge 的概率来源，供 graph、expansion、测试和序列化使用。

5.5 cognitive_graph.py

从 kernel 生成 cognitive graph，并在同一认知图层内完成 cost、directed distance、 symmetric distance 和 classical MDS。该模块表达的是当前数学模型中的认知几何，不是通用图算法库。

  • build_cognitive_graph(kernel, config)：从非零 kernel probability 生成 cognitive edges 和 -log(P) cost。
  • compute_directed_distances(graph)：小图默认使用 Floyd-Warshall；也可按 source 使用 Dijkstra。
  • symmetrize_distances(directed)：计算 (d(s,t)+d(t,s))/2。
  • compute_classical_mds(symmetric_distance, dim=2)：执行 classical MDS，输出每个 state 的二维坐标。
  • MDS 输入必须有限；包含 inf 或 NaN 时进入 explicit error。

5.6 planning.py

负责 cognitive path planning 和 cognitive edge expansion。 A* 核心函数只依赖 cognitive graph、embedding、start、goal 和 planning config； expansion 函数在 A* 输出 cognitive path 后，按表征类型把 edge 展开为 primitive trajectory。

  • plan_cognitive_path(graph, embedding, start, goal, config)。
  • successor 来自 cognitive graph 的 directed edges。
  • edge cost 使用 cognitive graph 中保存的 -log(P)。
  • heuristic 使用 alpha * euclidean_distance(embedding[start], embedding[goal])。
  • 输出 AStarResult，包括 path、path cost、expanded nodes、frontier diagnostics、status。

edge expansion 的 family-specific 规则如下：

family 候选来源 采样规则 展开结果
primitive 满足 T(s,a)=s' 的合法 action。 按 edge 条件下的 action weights 归一化采样。 单个 action 与 [s,s']。
chunk 满足 f(s,token)=s' 的 primitive/chunk token。 按 edge 条件下的 token weights 归一化采样。 token 对应的完整 primitive action/state trace。
landmark 原始任务图上从 s 到 s' 的 shortest primitive paths。 在所有 shortest paths 中均匀采样。 采样 path 的 action/state trace。

5.7 agent.py

提供面向调用方的统一载体 CognitiveNavigationAgent，负责持有 Representation、PlanningConfig 和最近一次构建的 ModelArtifacts。TaskGraph 是外部任务环境，不由 agent 拥有； 需要构建 artifact、搜索或运行 trial 时作为方法参数传入。Agent 是运行时门面，不承载 family-specific 数学逻辑； 具体计算仍委托给 kernel.py、cognitive_graph.py 和 planning.py。 初版不单独拆出 serialization.py，避免 artifact schema 和 run result 在两个模块之间来回跳转。

  • CognitiveNavigationAgent(representation, config)：创建带有某种认知表征的导航主体。
  • agent.build_artifacts(task)：对给定任务完成校验、kernel、graph、distance、embedding，并缓存 ModelArtifacts。
  • agent.plan(task, start_state, goal_state)：在给定任务环境中执行 cognitive graph 上的 A*。
  • agent.run_trial(task, trial_spec)：在给定任务环境中执行 A* 和 expansion，输出单 trial 结果。
  • agent.run_batch(task, trials)：批量运行，复用同一 task 下的 model artifacts。
  • to_dict(result) / to_json(result)：输出稳定 schema，保留中间 artifacts 与 trace。
Agent 解决的是“统一载体”和“调用入口”问题，不改变 kernel-first 的分层原则。 算法模块仍保持可单独测试，agent 只负责编排、缓存、错误包装和输出组织。

6. 核心数据模型

以下数据模型为设计契约，具体实现可用 dataclass、pydantic model 或等价结构。

6.1 TaskGraph

TaskGraph:
  task_id: str
  states: list[StateId]                 # 稳定顺序，决定矩阵 index 与 tie-breaking
  actions: list[ActionId]
  legal_actions: dict[StateId, list[ActionId]]
  transitions: dict[(StateId, ActionId), StateId]
  action_costs: dict[(StateId, ActionId), float]

约束：

  • states 不得为空，且不得包含重复值。
  • actions 不得包含重复值。
  • legal_actions[state] 中每个 action 必须属于 actions。
  • transitions[(state, action)] 只对合法 action 定义，target 必须属于 states。
  • action_costs[(state, action)] 必须为正数。

TaskGraph 是执行环境的完整定义，但第一版用户输入可以更简洁： 只显式输入嵌套字典形式的 transitions，其它字段由构造函数推导。 

TaskGraph.from_transitions(
  task_id: str,
  transitions: dict[StateId, dict[ActionId, StateId]],
  default_cost: float = 1.0,
  cost_overrides: dict[StateId, dict[ActionId, float]] | null = null,
) -> TaskGraph

推导规则：

  • states：所有 source state 和 target state 的并集，按稳定顺序排序或按输入顺序保留。
  • actions：所有 transition action key 的并集，按稳定顺序排序或按首次出现顺序保留。
  • legal_actions[state]：等于 transitions[state].keys()。
  • transitions[(state, action)]：由嵌套字典展开得到。
  • action_costs[(state, action)]：默认使用 default_cost，若 cost_overrides 指定则覆盖。

对带 loop 的九宫格任务，loop 只是普通 action label。 它不需要满足空间相邻关系，只需要在 transitions 中显式给出目标状态。

6.2 Token

PrimitiveToken:
  kind: "primitive"
  action: ActionId

ChunkToken:
  kind: "chunk"
  chunk_id: str
  actions: tuple[ActionId, ...]

primitive token 和 chunk token 统一作为 action-chunk 表征中的候选 token。 token 的 identity 必须稳定，序列化时使用 kind 与 id 区分。

6.3 Representation

PrimitiveRepresentation:
  representation_id: str
  family: "primitive"
  action_weights_by_state: dict[StateId, dict[ActionId, float]]

ChunkRepresentation:
  representation_id: str
  family: "chunk"
  chunk_inventory: dict[str, tuple[ActionId, ...]]
  token_weights_by_state: dict[StateId, dict[str, float]]

LandmarkRepresentation:
  representation_id: str
  family: "landmark"
  landmarks: list[StateId]
  p_ll: dict[StateId, dict[StateId, float]]
  p_lu: dict[StateId, dict[StateId, float]]
  p_ul: dict[StateId, dict[StateId, float]]

chunk 的 token_weights_by_state 中，primitive token 建议用规范 ID 表示，例如 primitive:right；chunk token 使用 chunk:g_rr。

RepresentationSpec:
  family: Family
  representation_id: str
  parameters: dict                       # 显式权重、chunk inventory、landmark set 或 CPD blocks
  defaults: dict                         # 可选默认规则，例如 uniform legal actions
  metadata: dict

RepresentationSpec 是 RepresentationBuilder 的输入，不是运行期使用的表征。 builder 读取 task 后必须输出上述三类 Representation 之一； agent 只接收已经构造完成的 representation。

6.4 TransitionKernel 与 KernelSupport

TransitionKernel:
  family: Family
  states: list[StateId]
  matrix: list[list[float]]              # shape = len(states) x len(states)
  row_index: dict[StateId, int]

KernelSupport:
  entries_by_edge: dict[(StateId, StateId), list[KernelSupportEntry]]

KernelSupportEntry:
  source: StateId
  target: StateId
  kind: "primitive_action" | "chunk_token" | "cpd_entry"
  item_id: str
  probability: float
  endpoint: StateId
  action_sequence: list[ActionId]
  state_sequence: list[StateId]
  cpd_block: "LL" | "LU" | "UL" | null

TransitionKernel 只表示状态转移概率矩阵。 KernelSupport 作为并列 artifact 保存概率来源，不内嵌在 TransitionKernel 中。 对 primitive/chunk，support entry 的 probability 是该 action/token 对 edge 概率的贡献。 对 landmark，support entry 的 probability 是 CPD entry 的概率，action_sequence 和 state_sequence 在 kernel 阶段为空，由 expansion 阶段填充。

6.5 CognitiveGraph

CognitiveGraph:
  family: Family
  states: list[StateId]
  edges: list[CognitiveEdge]
  successors: dict[StateId, list[StateId]]

CognitiveEdge:
  source: StateId
  target: StateId
  probability: float
  cost: float                            # -log(probability)

CognitiveEdge 只保存 edge 的概率和 cost；概率来源统一通过 KernelSupport 查询。

6.6 Distance 与 Embedding

DirectedDistance:
  states: list[StateId]
  matrix: list[list[float]]
  unreachable_pairs: list[(StateId, StateId)]

SymmetricDistance:
  states: list[StateId]
  matrix: list[list[float]]

Embedding:
  states: list[StateId]
  coordinates: dict[StateId, tuple[float, float]]
  eigenvalues: list[float]
  status: "ready"

本设计采用 fail-fast 策略：MDS 输入不有限时不生成 Embedding(status="ready")，而是抛出可识别错误。

6.7 ModelArtifacts

ModelArtifacts:
  task_id: str
  representation_id: str
  family: Family
  transition_kernel: TransitionKernel
  kernel_support: KernelSupport
  cognitive_graph: CognitiveGraph
  directed_distance: DirectedDistance
  symmetric_distance: SymmetricDistance
  embedding: Embedding

ModelArtifacts 是把某个 representation 应用到某个 task 后得到的可复用中间结果集合。 它不是输入表征，也不是单个 trial 的最终结果；agent 后续执行 plan 和 run_trial 时复用其中的 cognitive graph、embedding 和 kernel support。

6.8 TrialSpec、PlanningConfig 与 RunResult

TrialSpec:
  trial_id: str
  start_state: StateId
  goal_state: StateId
  family: Family
  random_seed: int | null

PlanningConfig:
  alpha: float = 0.0
  probability_epsilon: float = 1e-12
  row_sum_tolerance: float = 1e-9
  tie_breaking: "deterministic_state_order"
  max_landmark_shortest_paths: int = 10000
  mds_dim: int = 2

RunResult:
  status: "reached_goal" | "no_path" | "validation_error" | "embedding_error" | "expansion_error"
  task_id: str
  representation_id: str
  family: Family
  trial: TrialSpec
  artifacts: ModelArtifacts | null
  astar: AStarResult | null
  expansion: ExpansionResult | null
  metadata: RunMetadata
  error: ErrorDetail | null

alpha=0.0 表示 A* 退化为基于 cognitive graph cost 的 Dijkstra 行为； 当调用方希望使用 embedding heuristic 时，显式传入正数 alpha。

6.9 AStarResult 与 ExpansionResult

AStarResult:
  status: "found" | "no_path"
  start_state: StateId
  goal_state: StateId
  cognitive_path: list[StateId]
  path_cost: float | null
  expanded_nodes: list[StateId]
  visited_count: int
  tie_breaking: str

ExpansionResult:
  status: "reached_goal" | "expansion_error"
  cognitive_path: list[StateId]
  edge_expansions: list[ExpansionTrace]
  actions: list[ActionId]
  states: list[StateId]
  termination: TerminationRecord

ExpansionTrace:
  edge_index: int
  source: StateId
  target: StateId
  family: Family
  candidates: list[ExpansionCandidate]
  candidate_probabilities: list[float]
  sampled_index: int
  sampled_item_id: str
  actions: list[ActionId]
  states: list[StateId]
  reached_goal_at_step: int | null

6.10 CognitiveNavigationAgent

CognitiveNavigationAgent:
  representation: PrimitiveRepresentation | ChunkRepresentation | LandmarkRepresentation
  config: PlanningConfig
  cached_task_id: str | null
  cached_artifacts: ModelArtifacts | null

  build_artifacts(task: TaskGraph, force: bool = false) -> ModelArtifacts
  plan(task: TaskGraph, start_state: StateId, goal_state: StateId) -> AStarResult
  run_trial(task: TaskGraph, trial: TrialSpec) -> RunResult
  run_batch(task: TaskGraph, trials: list[TrialSpec]) -> list[RunResult]
  to_dict(result: RunResult) -> dict
  to_json(result: RunResult) -> str

Agent 的生命周期以一个 representation 和一份 config 为核心；task 是外部环境输入。 第一版只缓存最近一次 ModelArtifacts。若再次传入相同 task_id 且 force=false，agent 可以复用缓存；若 task 内容可能变化，调用方应传入 force=true 或创建新的 agent。

7. 核心算法设计

7.1 Primitive Kernel

for source in task.states:
    row = zeros(len(task.states))
    for action in task.legal_actions(source):
        target = task.next_state(source, action)
        p = representation.weight(source, action)
        row[target_index[target]] += p
        support[(source, target)].append(action_support(action, p))
    assert_normalized(row)
return TransitionKernel(matrix), KernelSupport(support)

如果多个 action 到达同一 target，edge probability 为这些 action 权重之和。

7.2 Chunk Kernel

for source in task.states:
    row = zeros(len(task.states))
    tokens = primitive_tokens_at(source) + executable_chunk_tokens_at(source)
    for token in tokens:
        trace = task.execute_token(source, token)
        target = trace.states[-1]
        p = representation.weight(source, token)
        row[target_index[target]] += p
        support[(source, target)].append(token_support(token, p, trace))
    assert_normalized(row)
return TransitionKernel(matrix), KernelSupport(support)

executable_chunk_tokens_at 必须逐步检查 chunk actions 是否全部合法。 任何无法从 source 完整执行的 chunk 都不得进入 token set，也不得参与归一化。

7.3 Landmark Kernel

L = set(representation.landmarks)
U = set(task.states) - L
for source in task.states:
    row = zeros(len(task.states))
    for target in task.states:
        if source in L and target in L:
            p = representation.p_ll[source][target]
            block = "LL"
        elif source in L and target in U:
            p = representation.p_lu[source][target]
            block = "LU"
        elif source in U and target in L:
            p = representation.p_ul[source][target]
            block = "UL"
        else:
            p = 0.0
            block = null
        row[target_index[target]] = p
        if p > epsilon:
            support[(source, target)].append(cpd_support(block, source, target, p))
    assert_normalized(row)
return TransitionKernel(matrix), KernelSupport(support)

self-transition 由 CPD 决定。若 CPD 给出正概率，就创建对应 cognitive edge。

7.4 Cognitive Graph 与 Cost

for source in states:
    for target in states:
        p = kernel[source, target]
        if p > probability_epsilon:
            cost = -log(p)
            add_edge(source, target, probability=p, cost=cost)
        else:
            no edge

probability_epsilon 只用于浮点噪声过滤。输入校验仍要求概率非负且行和在容差内等于 1。

7.5 Directed Distance

小规模任务默认使用 Floyd-Warshall，语义简单且便于测试。

dist = inf_matrix(n, n)
for s in states:
    dist[s, s] = 0
for edge in graph.edges:
    dist[edge.source, edge.target] = min(existing, edge.cost)
for k in states:
    for i in states:
        for j in states:
            dist[i, j] = min(dist[i, j], dist[i, k] + dist[k, j])

输出必须记录所有不可达 pair。对称化前不把 inf 替换成任意常数。

7.6 Classical MDS

assert all finite(D)
D2 = D ** 2
J = I - ones(n, n) / n
B = -0.5 * J @ D2 @ J
eigenvalues, eigenvectors = eigh(B)
take top dim eigenvalues in descending order
positive = maximum(eigenvalues, 0)
coordinates = eigenvectors[:, top] * sqrt(positive[top])

如果最大特征值包含轻微负值，按数值容差截断为 0； 如果输入矩阵不对称、对角线不为 0 或包含非有限值，返回 EmbeddingError。

7.7 A* 搜索

open_heap.push((f(start), h(start), 0.0, state_order[start], push_seq, start))
best_g[start] = 0
while open_heap:
    _, _, g, _, _, state = open_heap.pop()
    if stale_entry(state, g):
        continue
    if state == goal:
        return reconstruct_path(state)
    for next_state in graph.successors[state]:
        new_g = best_g[state] + graph.cost(state, next_state)
        if new_g < best_g.get(next_state, inf) - tolerance:
            update predecessor
            push with key (new_g + h(next_state), h(next_state), new_g, state_order[next_state], push_seq, next_state)
return no_path

tie-breaking 固定为 (f, h, g, state_order, push_seq)。 若同一节点出现数值相等的候选路径，保留先进入的 predecessor，保证结果可复现。

7.8 Edge Expansion

Primitive

candidates = [a for a in legal_actions(s) if T(s, a) == s_next]
probs = normalize([w_p(a | s) for a in candidates])
sampled = rng.choice(candidates, probs)
trace = actions=[sampled], states=[s, s_next]

Chunk

candidates = [token for token in tokens_at(s) if execute_token(s, token).endpoint == s_next]
probs = normalize([w_g(token | s) for token in candidates])
sampled = rng.choice(candidates, probs)
trace = execute_token(s, sampled)

Landmark

paths = task.all_shortest_paths(s, s_next, limit=config.max_landmark_shortest_paths)
if paths is empty:
    raise ExpansionError
sampled = rng.choice(paths, uniform)
trace = sampled

对 s == s_next，primitive shortest path 是零长度 path，展开为空 action sequence； 这与 CPD self-transition 的语义保持一致。

提前终止

for edge_index, edge in enumerate(cognitive_edges):
    trace = expand(edge)
    append trace step by step
    if any appended state == goal:
        record termination(edge_index, primitive_step_index)
        stop expansion

8. Agent 编排

8.1 创建 Agent

agent = CognitiveNavigationAgent(
    representation=representation,
    config=planning_config,
)

Agent 绑定一种认知表征和一份运行配置，不拥有任务环境。 调用方在构建 artifact、搜索或运行 trial 时传入 TaskGraph； agent 负责把 kernel、cognitive graph、distance、embedding、A* 和 expansion 串接起来。

8.2 build_artifacts

def CognitiveNavigationAgent.build_artifacts(task, force=False):
    if self.cached_artifacts is not None and self.cached_task_id == task.task_id and not force:
        return self.cached_artifacts
    validate_task_graph(task, self.config)
    validate_representation(task, self.representation, self.config)
    kernel, kernel_support = build_transition_kernel(task, self.representation, self.config)
    validate_kernel(kernel, self.config)
    graph = build_cognitive_graph(kernel, self.config)
    directed = compute_directed_distances(graph)
    symmetric = symmetrize_distances(directed)
    embedding = compute_classical_mds(symmetric, self.config)
    artifacts = ModelArtifacts(
        task_id=task.task_id,
        representation_id=self.representation.representation_id,
        family=self.representation.family,
        transition_kernel=kernel,
        kernel_support=kernel_support,
        cognitive_graph=graph,
        directed_distance=directed,
        symmetric_distance=symmetric,
        embedding=embedding,
    )
    self.cached_task_id = task.task_id
    self.cached_artifacts = artifacts
    return artifacts

8.3 plan

def CognitiveNavigationAgent.plan(task, start_state, goal_state):
    artifacts = self.build_artifacts(task)
    validate_start_goal(task, artifacts.cognitive_graph, start_state, goal_state)
    return plan_cognitive_path(
        artifacts.cognitive_graph,
        artifacts.embedding,
        start_state,
        goal_state,
        self.config,
    )

plan 只返回 cognitive graph 上的 A* 结果，不执行 primitive trajectory expansion。

8.4 run_trial

def CognitiveNavigationAgent.run_trial(task, trial):
    rng = make_rng(self.config, trial)
    artifacts = self.build_artifacts(task)
    astar = self.plan(task, trial.start_state, trial.goal_state)
    if astar.status == "no_path":
        return no_path_result(...)
    expansion = expand_cognitive_path(
        task,
        self.representation,
        artifacts.kernel_support,
        astar.cognitive_path,
        trial.goal_state,
        rng,
        self.config,
    )
    return reached_or_expansion_result(...)

8.5 run_batch

batch 运行复用同一 task 与同一 agent 组合下的 ModelArtifacts。 不同 trial 只执行 start/goal 校验、A* 和 expansion。 

def CognitiveNavigationAgent.run_batch(task, trials):
    self.build_artifacts(task)
    return [self.run_trial(task, trial) for trial in trials]

9. 校验与异常设计

9.1 校验阶段

阶段 校验内容 错误类型
TaskGraph 状态唯一、动作合法、transition 完整、target 属于状态集合、cost 为正。 InvalidTaskGraphError
PrimitiveRepresentation 权重非负，只覆盖合法 action，合法 action row sum 为 1。 InvalidPrimitiveRepresentationError
ChunkRepresentation chunk action 属于 action vocabulary；每个 state 的 executable token row sum 为 1。 InvalidChunkRepresentationError
LandmarkRepresentation landmark 属于状态集合；LL+LU row sum 为 1；UL row sum 为 1；概率非负。 InvalidLandmarkRepresentationError

Distance/MDS 对称距离有限、对称、对角线为 0。 EmbeddingError
Trial start/goal 属于状态集合；goal 在 cognitive graph 中可达。 PlanningError 或 NoPathResult

10.1 Trace 要求

  • 每个 nonzero kernel edge 必须能反查 support entries。
  • 每个 cognitive edge 必须能反查 probability 和 -log(P) cost。
  • 每次 expansion 必须记录 candidates、candidate probabilities、sampled index 和 sampled item。
  • 每个随机结果必须能通过 seed、候选顺序和采样概率复现。
  • 每个失败结果必须记录 stage、错误码、错误消息和关键上下文。

10.2 Candidate 顺序

为保证采样可复现，候选顺序必须稳定：

  • primitive action candidates 按 TaskGraph.actions 顺序。
  • chunk candidates 先列 primitive tokens，再列 chunk tokens；chunk tokens 按 chunk_inventory 的稳定顺序。
  • landmark shortest paths 按状态序列和动作序列的字典序排序后再采样。

11. 配置与随机性

11.1 配置对象

字段 默认值 含义
alpha 0.0 A* embedding heuristic scale。0 表示 zero heuristic。
probability_epsilon 1e-12 判断 kernel edge 是否存在的浮点阈值。
row_sum_tolerance 1e-9 概率行和校验容差。
distance_tolerance 1e-9 distance 对称性、对角线和数值比较容差。
max_landmark_shortest_paths 10000 landmark expansion 枚举 shortest paths 的上限。
tie_breaking deterministic_state_order A* tie-breaking 策略。
mds_dim 2 classical MDS 输出维度，当前需求为 2。

11.2 RNG 策略

  • 高层入口创建一个 RandomContext，所有 expansion 采样都通过它完成。
  • 若 TrialSpec.random_seed 存在，则优先使用 trial seed。
  • 若 trial seed 不存在，则使用 run config seed；若二者都不存在，结果仍可运行但 metadata 必须记录 seed=null。
  • RNG 对象不应在模块内部隐式创建。

12. 质量属性

属性 设计措施
正确性 每个数学对象对应一个显式数据模型；每个公式对应单独函数和测试入口。
可测试性 kernel、graph、distance、MDS、A*、expansion 均可用固定小图独立测试。

可审计性 RunResult 保存完整中间 artifact 和 trace，不只保存最终 actions/states。

性能 面向小规模研究图采用矩阵和清晰图算法；路径枚举设置上限。

15. 设计决策

决策 1
A* tie-breaking 使用固定 key：(f, h, g, state_order, push_seq)。状态顺序来自 TaskGraph.states。
理由
保证在相同输入、相同浮点结果下，搜索路径与 trace 可复现。
决策 2
alpha 是显式配置，默认 0.0，并写入 metadata。
理由
默认值对应 zero heuristic，保证规划仍按 cognitive graph cost 给出最短路径；需要 embedding heuristic 时由调用方显式指定。
决策 3
MDS 输入包含非有限 distance 时抛出 EmbeddingError，高层入口返回 embedding_error 状态。
理由
避免把不可达状态对静默填补成任意距离，保护 embedding 和 heuristic 的数值语义。
决策 4
landmark self-transition 按 CPD 处理。若 CPD 给出正概率，则 kernel 与 graph 中保留该 edge；expansion 返回零长度 primitive path。
理由
CPD 是 landmark kernel 的概率来源，软件层不额外修改其语义。
决策 5
landmark shortest primitive paths 默认全部枚举，受 max_landmark_shortest_paths 限制；超过上限时返回 ExpansionError。
理由
当前研究任务规模较小，枚举最清晰且能严格实现 uniform sampling；上限用于避免意外爆炸。
决策 6
CognitiveNavigationAgent 作为统一运行载体，持有 representation、config 和最近一次构建的 artifacts；不持有 TaskGraph。TaskGraph 作为外部任务环境传入 build_artifacts、plan、run_trial 和 run_batch。
理由
调用方需要一个稳定入口来执行完整流程；同时认知主体拥有的是表征而不是环境，算法仍保留在独立模块中，agent 只负责编排、缓存、错误包装和输出组织。
决策 7
从 task 和显式参数构造 representation 的职责属于 RepresentationBuilder，不属于 CognitiveNavigationAgent。
理由
同一 task 可以对应多种表征；表征包含模型假设和参数，不是 task 的唯一产物。把构造职责放在 builder 中，可以清晰区分“表征如何被设定”和“agent 如何使用表征执行认知导航”。
决策 8
TaskGraph 内部必须保存 legal_actions、transitions 和 action_costs；用户侧正式输入以 TaskGraph.from_transitions 的嵌套 transition 字典为准，其它字段由构造函数推导。
理由
当前模型的 token execution、kernel construction、A* edge expansion 和 primitive shortest paths 都依赖确定性转移函数 T(s,a)。直接输入 transitions 最简洁，也能表达 loop 这类非规则网格动作。
