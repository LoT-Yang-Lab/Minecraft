导航表征统一框架需求分析说明

基线文档：docs/开发文档/generative_model_macro_framework_cn.html

本文档基于当前宏观数学框架，定义软件系统需要支持的对象、功能、数据、校验规则、 输出结果和验收边界。当前需求的核心是 kernel-first：所有表征先转换为统一的状态转移核， 再共享 cognitive graph、directed distance、MDS embedding、A* 搜索和 edge expansion 流程。

1. 文档目的

本文档用于在进入软件设计和实现前，明确当前统一框架在软件层面的需求。 它描述“系统必须做什么”，不规定具体代码结构、类名、算法优化策略或文件组织。

本文档的输出对象包括：

  • 核心功能需求：从任务与表征生成 transition kernel，并进一步生成认知图、距离、地图和 A* path。
  • 三类表征需求：primitive、action-chunk、landmark-state。
  • 执行展开需求：把 cognitive edge 转换为实际 primitive trajectory。
  • 数据与结果需求：中间对象和最终输出必须可审计、可序列化、可测试。
  • 边界需求：当前数学框架已定义的内容、尚未定义的内容，以及进入软件设计前需要明确的问题。

2. 需求基线与核心链条

当前数学框架的主链条为：

(K0, RM) → PM(s' | s) → cM(s,s') → dM→(s,t) → DM(s,t) → ψM(s) → A* → edge expansion
当前框架的统一接口是 row-stochastic transition kernel。 表征类型差异体现在 RM → PM； 一旦得到 PM，后续 cost、distance、MDS、A* 和路径展开共享同一套流程。

3. 范围与非范围

3.1 当前版本范围内

  • 定义 deterministic primitive navigation task graph。
  • 定义 primitive、action-chunk、landmark-state 三类静态表征。
  • 从每类表征构造 row-stochastic transition kernel。
  • 从 transition kernel 构造 directed cognitive graph 和 surprisal edge cost。
  • 计算 directed shortest distance。
  • 对称化 directed distance，得到 MDS 输入矩阵。
  • 执行 classical MDS，得到二维 cognitive map embedding。
  • 在 cognitive graph 上执行 A* 搜索。
  • 把 A* 输出的 cognitive path 展开为 primitive action / state trajectory。
  • 输出完整中间结果和最终轨迹，支持后续分析与测试。

3.2 当前版本范围外

  • 表征学习、参数拟合和模型反推流程不属于本文档定义的正向 pipeline。
  • landmark region membership 不属于当前 landmark-state 表征定义；当前 landmark 是具体 state set。
  • stochastic environment transition 不属于当前任务定义；任务转移函数 T(s,a) 是确定性的。
  • 交互式可视化 UI 不属于当前需求；MDS embedding 只要求作为数据 artifact 输出。
  • 二维 embedding 不是搜索图；A* 搜索始终在 cognitive transition graph 上进行。

4. 用户与使用场景

角色 需求
研究者 比较不同导航表征诱导出的 transition kernel、认知距离、地图和规划路径。
模型开发者 实现和验证从表征到 kernel，再到 A* path 和展开轨迹的完整流程。
测试维护者 使用固定小图和 golden case 验证 kernel、cost、distance、A*、edge expansion 是否符合数学定义。
后续推断模块开发者 读取完整中间数据，用于未来 likelihood、模型比较或反推表征。

5. 术语与对象

术语 含义 软件需求含义
K0 primitive task graph。 输入任务对象，包含状态、action vocabulary、合法动作、确定性转移和 primitive cost。
RM family M 的静态表征。 输入表征对象；必须能构造 PM。
PM(s'|s) 由表征诱导的状态转移核。 核心统一接口；每一行必须归一化。
cM(s,s') 有向边 cost，等于 -log PM(s'|s)。 cognitive graph 上的 A* edge cost。
dM→ 在 directed cost graph 上的最短距离。 用于构造对称距离和分析表征几何。
DM 对称化距离矩阵。 MDS 输入。
ψM 二维 cognitive map embedding。 用于可视化和 A* heuristic，不是搜索图。
cognitive path A* 在 cognitive graph 上输出的状态序列。 需要进一步展开为 primitive trajectory。
edge expansion 把 cognitive edge 展开成 primitive action / state sequence。 family-specific 的执行阶段。

6. 总体业务流程

  1. 加载并校验 primitive task graph K0。
  2. 加载并校验 family-specific representation RM。
  3. 根据 family 构造 transition kernel PM。
  4. 从 PM 构造 cognitive graph：边集合和 -log P cost。
  5. 计算 directed shortest distance dM→。
  6. 对称化得到 DM，并执行 MDS 得到 ψM。
  7. 对给定 trial 的起点和目标，在 cognitive graph 上执行 A*。
  8. 将 A* 输出的 cognitive path 逐 edge 展开为 primitive trajectory。
  9. 如果展开过程中提前到达 trial goal，立即终止后续展开。
  10. 输出 observed trajectory、cognitive path 和所有中间对象。

7. 功能需求

FR-1 TaskGraph 需求

ID 需求
FR-1.1 系统必须表示有限状态集合 S。
FR-1.2 系统必须表示 primitive action vocabulary A。
FR-1.3 系统必须表示每个状态下的合法 action 集合 A(s)。
FR-1.4 系统必须提供确定性转移函数 T(s,a)，仅对合法 action 定义。
FR-1.5 系统必须支持 primitive action cost κ(s,a)>0。
FR-1.6 系统必须能计算 primitive shortest-path distance d*(s,t)，供 landmark edge expansion 使用。

FR-2 Representation 需求

FR-2P Primitive representation

  • FR-2P.1：系统必须支持 RP=(A,wP)。
  • FR-2P.2：wP(a|s) 只对 a in A(s) 参与归一化。
  • FR-2P.3：对每个状态 s，合法 action 的权重和必须为 1。

FR-2C Action-chunk representation

  • FR-2C.1：系统必须支持 chunk inventory G，每个 chunk 是 primitive action 序列。
  • FR-2C.2：系统必须能为每个状态计算可完整合法执行的 chunk 集合 G(s)。
  • FR-2C.3：系统必须定义 token set TG(s)=A(s) union G(s)。
  • FR-2C.4：系统必须支持 wG(token|s)，并保证在 token set 上归一化。
  • FR-2C.5：系统必须能计算任意可执行 token 的完整执行终点 f(s,token)。

FR-2L Landmark representation

  • FR-2L.1：系统必须支持 landmark state set L subset S，且去重后数量至少为 2。
  • FR-2L.2：系统必须定义 non-landmark set U=S\L。
  • FR-2L.3：系统必须支持 CPD block pLL(l'|l)。
  • FR-2L.4：系统必须支持 CPD block pLU(u|l)。
  • FR-2L.5：系统必须支持 CPD block pUL(l|u)。
  • FR-2L.6：每个 landmark source row 中 pLL 与 pLU 联合归一化为 1。
  • FR-2L.7：每个 non-landmark source row 中 pUL 归一化为 1。
  • FR-2L.8：当前版本不允许 non-landmark 到 non-landmark 的直接认知转移；对应 kernel block 为 0。

FR-3 Transition Kernel 需求

  • FR-3.1：系统必须为每个 family 构造 PM(s'|s)。
  • FR-3.2：每一行 PM(.|s) 必须在状态集合 S 上归一化。
  • FR-3.3：primitive kernel 必须按支持同一终点的 action 权重求和。
  • FR-3.4：chunk kernel 必须按支持同一终点的 primitive/chunk token 权重求和。
  • FR-3.5：chunk kernel 构造时只允许使用完整合法执行的 chunk。
  • FR-3.6：landmark kernel 必须通过 CPD blocks 查表拼接，不得从 landmark set 自动推导概率。
  • FR-3.7：系统必须保留 kernel 的 provenance：每个 kernel edge 由哪些 action/token/CPD entry 支持。

FR-4 Cognitive Graph 与 Cost 需求

  • FR-4.1：系统必须从 PM(s'|s)>0 构造 directed edge set。
  • FR-4.2：系统必须为每条 directed edge 计算 cM(s,s')=-log PM(s'|s)。
  • FR-4.3：当 P=0 时，不应创建 edge；查询 cost 时可返回 infinity 或明确表示不可达。
  • FR-4.4：系统必须保留 edge probability 和 edge cost，便于调试和测试。

FR-5 Directed Distance 与 MDS 需求

  • FR-5.1：系统必须计算 directed shortest distance dM→(s,t)。
  • FR-5.2：系统必须支持检测 directed unreachable pairs。
  • FR-5.3：系统必须按 D(s,t)=(d(s,t)+d(t,s))/2 构造对称距离。
  • FR-5.4：若对称距离矩阵包含 infinity 或 NaN，系统必须显式报错或返回 partial result；不得静默传入 MDS。
  • FR-5.5：系统必须支持 classical MDS 输出二维 embedding psi(s)。

FR-6 A* 搜索需求

  • FR-6.1：系统必须在 cognitive graph 上执行 A*，而不是在 MDS embedding graph 上执行。
  • FR-6.2：A* successor 必须来自 PM(s'|s)>0 的 edge set。
  • FR-6.3：A* edge cost 必须使用 -log P。
  • FR-6.4：A* heuristic 必须由 embedding distance 给出：alpha * ||psi(s)-psi(goal)||。
  • FR-6.5：系统必须支持配置或记录 alpha。
  • FR-6.6：系统必须输出 cognitive path、visited/search diagnostics 和最终 path cost。
  • FR-6.7：若 goal 在 cognitive graph 上不可达，系统必须返回明确的 no-path 结果或抛出可识别错误。

FR-7 Cognitive Edge Expansion 需求

Primitive edge expansion

  • FR-7P.1：对 cognitive edge (s,s')，系统必须找到所有满足 T(s,a)=s' 的 action。
  • FR-7P.2：若有多个 action 支持同一 edge，系统必须按条件归一化后的 wP 采样。
  • FR-7P.3：展开结果必须是一条单步 primitive trajectory。

Action-chunk edge expansion

  • FR-7C.1：对 cognitive edge (s,s')，系统必须找到所有满足 f(s,token)=s' 的 token。
  • FR-7C.2：若多个 token 支持同一 edge，系统必须按条件归一化后的 wG 采样。
  • FR-7C.3：若采样 token 是 chunk，系统必须按任务转移函数展开完整 primitive action sequence。

Landmark edge expansion

  • FR-7L.1：对 landmark cognitive edge (s,s')，系统必须在原始任务图中寻找所有 shortest primitive paths。
  • FR-7L.2：系统必须从这些 shortest paths 中均匀采样一条。
  • FR-7L.3：若不存在 primitive path，系统必须报错；CPD 不应指向环境不可达的展开目标，除非后续明确允许 partial execution。

通用终止规则

  • FR-7G.1：展开 cognitive path 时，如果实际状态序列提前到达 trial goal，系统必须立即终止 trial。
  • FR-7G.2：终止后不得继续展开后续 cognitive edges。
  • FR-7G.3：系统必须记录终止发生在哪个 cognitive edge 和哪个 primitive step。

8. 数据需求

8.1 输入数据

对象 必需字段
TaskGraph states、action vocabulary、legal actions by state、transition function、primitive costs。
PrimitiveRepresentation action weights per state。
ChunkRepresentation chunk inventory、token weights per state。
LandmarkRepresentation landmark set、LL/LU/UL CPD blocks。
TrialSpec trial id、start state、goal state、family id、optional random seed。
PlanningConfig A* heuristic scale alpha、tie-breaking policy、random seed、MDS settings。

8.2 中间数据

  • TransitionKernel：矩阵或稀疏行结构。
  • KernelSupport：每条 kernel edge 的 action/token/CPD 来源。
  • CognitiveGraph：edges、probabilities、costs。
  • DirectedDistance：directed shortest-distance matrix。
  • SymmetricDistance：MDS 输入矩阵。
  • Embedding：每个 state 的二维坐标。
  • AStarTrace：open/closed diagnostics、expanded nodes、path cost、final cognitive path。
  • ExpansionTrace：每条 cognitive edge 的 family-specific expansion 结果。

8.3 输出数据

  • Observed trajectory：状态序列和 primitive action 序列。
  • Cognitive path：A* 输出的状态路径。
  • Expanded edges：每条 cognitive edge 对应的实际 primitive path。
  • Model artifacts：kernel、cost graph、distances、embedding。
  • Run metadata：family、representation id、trial id、random seed、配置参数、状态。

9. 校验与异常需求

ID 校验规则 失败处理
VAL-1 TaskGraph 的 transition source/target/action/cost 必须一致且合法。 返回 invalid task graph error。
VAL-2 所有 probability 或 weight 必须非负。 返回 invalid representation error。
VAL-3 每个 state 的 primitive weights 必须在合法 action 上归一化。 返回 invalid primitive representation error。
VAL-4 chunk weight row 必须在 executable token set 上归一化。 返回 invalid chunk representation error。
VAL-5 landmark CPD blocks 必须满足 LL+LU row sum = 1，UL row sum = 1。 返回 invalid landmark representation error。
VAL-6 构造出的 TransitionKernel 每行必须归一化。 返回 invalid kernel error。
VAL-7 MDS 前的对称距离矩阵必须有限，或系统必须进入 explicit partial/blocked 状态。 返回 embedding blocked error 或 partial result。
VAL-8 A* start/goal 必须属于状态集合，且 goal 在 cognitive graph 中可达。 返回 no-path result 或 planning error。
VAL-9 landmark edge expansion 的 source/target 必须在 primitive task graph 中可达。 返回 expansion error。

10. 输出与可追溯性

系统输出必须支持从最终轨迹反查到每个阶段的决策依据。

{
  "task_id": "...",
  "family": "primitive | chunk | landmark",
  "representation": {...},
  "transition_kernel": {...},
  "kernel_support": {...},
  "cognitive_graph": {
    "edges": [...],
    "costs": [...]
  },
  "directed_distance": {...},
  "symmetric_distance": {...},
  "embedding": {...},
  "trials": [
    {
      "trial_id": "...",
      "start_state": "...",
      "goal_state": "...",
      "cognitive_path": [...],
      "edge_expansions": [...],
      "states": [...],
      "actions": [...],
      "status": "reached_goal | no_path | expansion_error | partial"
    }
  ]
}
  • 每条 cognitive edge 必须能查到 probability、cost 和 support 来源。
  • 每次 edge expansion 必须记录采样候选、采样概率、sampled item 和展开后的 primitive path。
  • 若提前到达 goal，必须记录提前终止位置。
  • 若某阶段失败，必须记录失败阶段和原因。

11. 非功能需求

类别 需求
可复现性 所有随机采样必须由显式 seed 或 rng 控制。
数值稳定性 概率归一化、-log(P) 和 MDS 输入必须处理浮点容差。
可测试性 每个 pipeline stage 必须能单独测试。
可审计性 不得只输出最终轨迹；必须输出关键中间对象。
模块化 表征到 kernel 的 family-specific 逻辑必须与 shared graph/distance/A* 逻辑分离。
性能 当前以小规模研究任务为目标；实现应优先清晰正确，避免过早优化。

12. 验收标准

  • 给定合法 primitive representation，系统能生成 row-stochastic primitive kernel。
  • 给定合法 chunk representation，系统能过滤 executable chunks，并正确汇总同终点 token 概率。
  • 给定合法 landmark representation，系统能从 LL/LU/UL CPD blocks 拼接 landmark kernel，且 U-U block 为 0。
  • 系统能把任意合法 kernel 转换为 cognitive graph 和 -log(P) cost。
  • 系统能计算 directed distance，并显式处理不可达状态对。
  • 系统能对有限 symmetric distance 执行 MDS，输出二维 embedding。
  • 系统能在 cognitive graph 上执行 A*，输出 cognitive path。
  • 系统能按 family-specific 规则展开 cognitive edge。
  • 系统能输出完整中间 artifacts 和最终 trajectory。
  • 所有核心需求都有对应单元测试、集成测试或 golden case。

13. 待决问题

以下问题在宏观文档中尚未完全细化，进入详细设计前需要明确或以显式默认值处理：

ID 问题 建议处理
OPEN-1 A* tie-breaking policy 未定义。 详细设计中固定 deterministic tie-breaker，并记录到 metadata。
OPEN-2 MDS 如何处理 infinite distance 未定义。 当前需求要求显式报错或 partial result；不静默填补。
OPEN-3 heuristic scale alpha 如何设定未定义。 作为配置参数输入，并输出到 run metadata。
OPEN-4 landmark CPD 是否允许 self-transition 未明确。 详细设计中需要定义；当前需求只要求若存在概率则按 CPD 处理。
OPEN-5 landmark edge 的 shortest primitive paths 若数量很大，是否枚举全部未定义。 当前小规模任务可枚举；后续大图可替换为等价采样算法。
